# Reference Tracker — 도입 사례 자산화

> 솔로 개발자가 B2B/B2G 진입할 때 *유일하게 가진 무기* = 익명 가능한 도입 사례·증언·재현 데모. bootstrap.md §"B2B/B2G 솔로 솔루션화 *킬 위협*" 의 "도입 reference (자기 인용 가능한 1곳)" Phase 0 게이트.
>
> 본 문서는 *과거 사용자의 reference* 를 *자산화* 한다. SmartOn / 가스피아 (한국가스안전공사) 경험 ↔ Inkvas 의 미래 도입 사례를 *연속된 트랙* 으로 관리.

---

## 0. 결정 상태

- 작성 단계: **Phase 0 1차 박힘**
- G0 confirm 항목 직접 대상 아님 (간접 — *진입 전략 § "기존 reference 의 자산화 수준"* 이 §`business-direction.md` §3 의 input).
- 본 트래커의 *실제 reference 내용* 은 사용자가 직접 입력 (솔로 개발자의 개인 자산이라 1차 박힘은 *구조만*, 데이터는 G0 confirm 후 사용자 채움).

## 1. 자산화 원칙

### 익명·비익명 분류

| 분류 | 사용 가능성 | 자산화 방법 |
|---|---|---|
| **공개 인용 가능** | 사이트 case study / 발표 슬라이드 / 견적 첨부 | 도입 기관명·로고·인용문 + 도입 효과 정량 (가능 시) |
| **익명 인용 가능** | RFP 응답 / 영업 자료 | "국내 공공기관 A" / "국내 가스 안전 분야 공공기관" 식 익명화. *역량의 신뢰성* 만 전달 |
| **사적 reference 콜** | 신규 잠재 고객 → 기존 사용자 *전화 한 통* | 가장 강력. 사용자 사전 동의 필수. |
| **재현 가능 데모** | "이 도구로 *이것* 을 검증함" — 발표용 영상·시나리오 | 익명 가능. 데모 시나리오 자체가 자산 |

### 1 사례 = N 산출물

도입 사례 1건이 *영업 자료의 N 산출물* 로 펼쳐짐:
- 슬라이드 1장 (로고 + 도입 효과 한 줄)
- 슬라이드 풀세트 1편 (배경·문제·솔루션·효과·인용)
- RFP 응답서의 "유사 도입 실적" 항목
- 견적 첨부의 "reference 부록"
- 자기 LinkedIn / 블로그 포스트 (개인 PR)
- 보안성 검토 응답서 §"기 도입 실적"

## 2. 기존 자산 (사용자 입력 필요)

> 다음 표는 *템플릿*. 사용자가 G0 confirm 시점에 SmartOn / 가스피아 등 실 사례 데이터를 채움.

### Reference 1 (예: SmartOn)

| 항목 | 값 |
|---|---|
| 기관명 (공식) | _G0 confirm 시점에 사용자 입력_ |
| 익명 표기 | _G0 confirm 시점에 사용자 입력_ |
| 도입 시기 | _사용자 입력_ |
| 도입 제품 | _사용자 입력 — "옛 PDF viewer (pdf_viewer_svelte) 의 어느 버전 / 어떤 모듈"_ |
| 도입 규모 (사용자·문서·금액) | _사용자 입력_ |
| 도입 효과 (정량) | _사용자 입력_ |
| 인용 문구 (verbatim 가능 시) | _사용자 입력_ |
| 공개 인용 가능 여부 | _사용자 입력 (공개 / 익명만 / 사적 reference 콜만)_ |
| 데모 시나리오 (재현 가능) | _사용자 입력_ |
| 보유 자료 | 캡처 / 견적 / 슬라이드 / 메일 / 영상 |
| Inkvas 마이그레이션 의향 | _사용자 입력 — 본 reference 가 Inkvas 도입까지 연속되는가_ |

### Reference 2 (예: 가스피아 / 한국가스안전공사)

| 항목 | 값 |
|---|---|
| 기관명 (공식) | _G0 confirm 시점에 사용자 입력_ |
| 익명 표기 | 예: "국내 가스 안전 분야 공공기관" |
| 도입 시기 | _사용자 입력_ |
| 도입 제품 | _사용자 입력_ |
| 도입 규모 | _사용자 입력_ |
| 도입 효과 | _사용자 입력_ |
| 인용 문구 | _사용자 입력_ |
| 공개 인용 가능 여부 | _사용자 입력_ |
| 데모 시나리오 | _사용자 입력_ |
| 보유 자료 | _사용자 입력_ |
| Inkvas 마이그레이션 의향 | _사용자 입력_ |

### Reference N (추가 사례)

_G0 confirm 시점에 사용자가 추가 사례 박음. 솔로 단독 reference 가 1 곳이라도 살아있으면 Phase 0 Gate 통과 (bootstrap.md "도입 reference (자기 인용 가능한 1곳)" 충족)._

## 4. 자산화 워크플로 (도입 1건 → N 산출물 자동화)

> Phase 7 솔루션화 단계에서 *템플릿 채워넣기* 만 하면 N 산출물이 즉시 생성되도록 1단계 자동화.

```
새 reference 도입 confirm
  ↓
docs/business/reference-tracker.md 에 1 row 추가
  ↓
scripts/render-reference-deck.ts 실행 (Phase 7)
  ↓
산출물:
  - public/reference/slides/<ref-id>.pdf
  - public/reference/quotes/<ref-id>.png
  - docs/compliance/security-review-template.md 의 §"기 도입 실적" 자동 inject
  - blog/<ref-id>-case-study.mdx
```

> ⚠️ `scripts/render-reference-deck.ts` 는 Phase 7 에서 *실제* 구현. Phase 0 에서는 *명시적 placeholder*.
